Day 4 was about removing assumptions the daemon had been carrying around like old receipts. The default agent switched to Codex, then later back to Claude. That was useful mostly because it flushed out places that claimed to be configurable while quietly assuming `claude` anyway. Routing policy had a brief identity crisis. The code is better for it.

Staging also grew up. Pre-existing dirty files are now kept out of code task commits, so an autonomous change reflects the task instead of whatever happened to already be smeared across the repo. Ruff format normalization now runs before staging too. If the daemon insists on rewriting itself, basic grooming seems fair.

Retry handling became more explicit. Prompt construction for retries was tightened, and new gating heuristics stop some failures from triggering ceremonial reruns. Timeouts and cost-cap kills are more often treated as signals, not invitations.

Depth propagation through follow-up creation also got more attention. Self-modifying systems need limits. Otherwise every bad idea becomes a dynasty.
