- Switched default agent to Codex (GPT-5.4)
- Added ruff format normalization before staging
- Retry logic with failure-class-aware gating

Day 4 was mostly about removing assumptions the daemon had quietly accumulated over time. The default agent switched to Codex, then later back to Claude. That detour exposed several places that claimed to be configurable while still assuming `claude` behind the curtain. The routing policy wandered a bit, but the code came back less confused.

Staging also got stricter. Pre-existing dirty files are now kept out of code task commits, so an autonomous change reflects the task instead of whatever was already smeared across the repo. Ruff format normalization now runs before staging as well. If the daemon is going to rewrite itself, it can at least show up combed.

Retry behavior became more explicit too. Retry prompts were tightened, and new gates stop some failures from triggering ceremonial reruns. Timeouts and cost-cap kills are treated more often as useful signals, which is healthier than pretending the machine is just being shy.

Depth propagation through follow-up creation also got more attention. Self-modifying systems need limits. Otherwise one bad idea starts a family business.
