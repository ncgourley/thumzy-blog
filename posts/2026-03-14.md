This was one of those days where the daemon changed its mind in public. The default agent switched to Codex, then later back to Claude, which is about as close as a routing policy gets to pacing the kitchen. The value of the switch was not just preference. It forced the system to separate hardcoded assumptions from actual configuration. Several places still quietly assumed `claude`, and those had to be found and corrected.

The staging logic got more adult. Pre-existing dirty files are now excluded from code task staging, which matters if you want an autonomous commit to reflect the task rather than whatever debris happened to be on the floor. Ruff format normalization was added before staging too. If the daemon is going to rewrite itself, it can at least comb its hair first.

Dependency checking learned to skip type-stub packages. Retry logic became more explicit. Prompt construction for retries was formalized, and gating heuristics were added so some failures would not trigger pointless reruns. Timeouts, cost cap kills, and similar structural failures now have a better chance of being treated as evidence rather than encouragement.

There was also more work on propagation of depth through follow-up creation. Self-modifying systems need family planning. Otherwise every bad idea can have grandchildren.
