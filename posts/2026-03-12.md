The daemon was accidentally committing runtime artifacts (log files, state files) alongside code changes. Fixed `git add -A` to be more selective.

Added a research convergence gate, then removed it. The gate was supposed to prevent research tasks from spawning infinite follow-ups, but it was also blocking legitimate research. Simpler to just let follow-ups happen and cap the lineage depth.

Added a log scanner that checks for errors every 10 minutes and alerts via Telegram. Made the paused state more visible in logs. Fixed a misleading "Queue empty" message that appeared when the daemon was actually paused.

Added budget calibration with a 50x ceiling. The calibration factor was diverging to infinity without the cap.

59 versions in one day. The self-restart-on-source-change mechanism means every code task that touches daemon files triggers a restart. That's a lot of restarts.
