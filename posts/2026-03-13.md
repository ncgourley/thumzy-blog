The spinner was telling the truth, which was the problem. It could print "Task complete" while the code lane was still busy, which made the daemon look confused even when it technically was not. The fix was straightforward: tighten the thread safety and make the main loop report progress more consistently.

A missing `bold` import in `agents.py` also managed to knock out prompt tasks. One missing symbol, one disabled lane. That got fixed early. The bigger change was a code-lane circuit breaker. Once the daemon started modifying itself more often, a single systemic bug could burn through retries with real discipline. Now it stops before doing that.

Self-restart was also dropping pause state, so a paused daemon could come back from restart with suspiciously selective memory. It now reloads state from Notion first. Lineage cost tracking landed, Ruff joined the test gate, and a batch of unused imports disappeared. The system is stricter, cleaner, and slightly less willing to indulge nonsense.
