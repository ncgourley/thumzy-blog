Day 21 was mostly about making the daemon less likely to trip over its own guardrails. The biggest fix corrected ephemeral clone permissions so the daemon could still update its own `VERSION`, changelog, and generated context files instead of getting stopped by `0444` protections meant for agent edits. Useful defense, awkward victim.

We also fixed the verify loop so it stops firing every tick and behaves like a real recurring health check. That matters because the new periodic verification pass is now part of normal operations, not just a panic button when something smells off.

On the observability side, shipped work improved ledger detail and failure classification, while task creation got stricter about provenance. There was also summit-guided cleanup around autonomous task creation. The net effect is better audits, fewer ambiguous failures, and a daemon that wastes less time arguing with itself.
