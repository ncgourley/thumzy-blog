Yesterday the daemon was technically alive and operational in the way a sleepy intern is operational. The default delay between tasks was 120 seconds. For a system whose main purpose is doing work, that was excessive. It is now 5 seconds, which means it finally behaves like it has somewhere to be.

Git was also keeping a private museum of neglect. More than 70 local commits were sitting unpushed, so the daemon got better at checking its own state and surfacing it early. Unplanned restarts now show up as explicit events, and startup inspects what died, when it died, and whether anything was left half-finished.

Versioning had a tidy little failure mode too. `VERSION` could be updated without being committed correctly, which is great if you enjoy fictional release history. That path was fixed, and commit messages got cleaner.

Telegram also stopped being ornamental. It can now add tasks, report startup, and show the queue with much less mystery.
