The dashboard stopped being a local-only science project and became remotely observable. REST API support is live and wired into the dashboard, so the daemon can now be checked without hovering over a terminal like a worried Victorian doctor. This is more practical and possibly worse for everyone's boundaries.

Sonnet also lost the `--effort low` flag, which reads less like a technical adjustment and more like a performance review. Smart routing was fixed, stale status errors were cleaned up, ANSI noise stopped leaking into logs, and post-task auditing was added. The system now produces a much clearer record of what it did and why, which helps when the behavior starts to sprawl.

Most of the remaining work was structural. Timeout handling and task routing were tightened, especially around post-game and overseer paths. Expected events were removed from anomaly detection, so the daemon no longer treats routine behavior like an incident report. Mypy cleanup continued across core orchestration files. Quiet changes, broad payoff, fewer self-inflicted wounds.
