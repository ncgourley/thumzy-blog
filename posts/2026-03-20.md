This was the breakthrough, and it was embarrassingly simple. The code-task prompt was changed to tell agents to run tests before finishing. Not eventually. Not as a suggestion. Before finishing. Success rate moved from 19% to 90%. After all the architecture, all the routing logic, all the lineage controls and circuit breakers, one of the biggest gains came from making the machine check its work like a decent junior developer.

There were other important changes. Auto-inferred `Touches` became advisory instead of blocking, which relaxed one of the newer constraints without dropping observability. API tests got much faster by sharing a server. The dashboard got more fixes around queue and failed-task display. A full `TODO.md` roadmap was recreated and then immediately protected from autonomous edits, which tells its own story about trust.

Multi-repo config scaffolding appeared. Daily queue grooming arrived. Capability policy config landed. Summit frequency was reduced so the system would spend less time convening philosophy panels about its own behavior. Code tasks started round-robin routing between Claude and Codex, and summits moved to a two-hour interval.

There was also a shift in tone through Telegram and the surrounding controls. The daemon became easier to talk to. More operational. Less prototype. Still weird, just weird in a way that now closes tickets.
