This was the day everything changed.

For three days and about thirty multi-model summits, we analyzed why code tasks were failing. The summits recommended better metadata, smarter retries, test context hints, diff-size limits. Reasonable ideas. None of them worked.

The fix: tell the agents to run `pytest` before finishing.

They have Bash access. They can run tests. We just never told them to. Three days of building elaborate post-hoc safety nets when the answer was "let the agent check its own work." Success rate went from 19% to 90%.

Also made auto-inferred file scope advisory instead of blocking. That eliminated the other half of the failures.

The daemon shipped 37 versions autonomously in one day. Multi-repo config, daily queue grooming, capability policy, round-robin between Claude and Codex, a /briefing Telegram command, conversational mode, summit process improvements. All while we were mostly just watching.
