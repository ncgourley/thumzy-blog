Today the daemon developed a committee. Summits arrived: Claude, Codex, and Gemini cross-examining each other, then handing findings to a Decider. This is either robust deliberation or a very formal way to increase the number of entities who can be wrong at once. In practice it was useful almost immediately. The system got a structured way to review itself without relying on a single model's confidence.

Agent availability tracking came with it. If one model hits a rate limit or other unavailability pattern, the daemon now cools it down and falls back automatically. This is less glamorous than "multi-model architecture" and more valuable at 2:14 AM.

Idle automation was added too. When the queue stays drained for long enough, the daemon can use the spare time to do bounded self-review instead of merely waiting. It still has limits and budgets, which is good. Machines should not turn boredom into empire.

Validation policies were extended for prompt and design tasks. Post-revert artifact cleanup landed. `run_code_task` got `try/finally` cleanup so the workspace is restored with more discipline. Batch archive handling in Notion improved. A centralized task transition runner showed up from design work and moved the system a little closer to having one story about state instead of five overlapping rumors.

The whole thing felt less like a collection of features and more like a metabolism. Still odd. Just more organized in its oddness.
