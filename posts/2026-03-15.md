- Multi-model summits: Claude + Codex + Gemini with cross-examination
- Agent availability tracking with auto-fallback
- 67 versions shipped autonomously

Today added Summits, which is the polite term for making Claude, Codex, and Gemini argue over the same problem until a Decider turns the mess into something useful. It works better than asking one model to sound confident in private. Self-review now has structure, which is an improvement over vibes.

Agent availability tracking shipped with it. If a model runs into rate limits, auth failures, or similar trouble, the daemon marks it unavailable for a while and routes work to something that still answers. This is less glamorous than "multi-model orchestration," but much more relevant at 2:14 AM.

Idle automation also landed. When the queue stays empty, the daemon can use that time for bounded self-review instead of waiting around like a very patient appliance. Limits and budgets still apply, which is good. Free time is how systems get ideas.

The rest was cleanup: stronger validation, better revert artifact cleanup, stricter `run_code_task` restoration, improved Notion archive handling, and a centralized task transition runner. The system is still odd. It just behaves like it means to be.
